About

I decided to make the career change to become a web developer, in order to push myself and give myself the creativity to create amazing things using only my two hands and a computer. From a young age, when I used to spend hours tinkering with the HTML and CSS on my myspace profile page, I always found some enjoyment in coding, and technology in general, so after some deliberation and discussion with my friends in the industry, I decided to finally make the jump into coding by joining General Assembly, and I couldn’t be happier.

I have have really enjoyed the challenges and wins that coding has already provided me, and I eagerly await the next ones. I am looking forward to beginning my career as a developer in an environment that not only is full of friendly people, but also gives me a great opportunity to solve challenging problems with others, learn new technology and methods, and also to become a more accomplished developer.

My Work

Project 1: Battleship

My first project three weeks into the course at GA was grid based battleship game, worked on over seven days for a solo project. I used jQuery to manipulate much of the game board, and used CSS/SCSS along with HTML audio to give the game a unique style.

Tech Used: HTML/CSS, SCSS, JavaScript, jQuery

Project 2: Movie Library

My second project five weeks into the course was a two day hackathon, accomplished with a partner. It is a site designed to store your movie library and give you recommendations on new movies to watch. We used React to build out the front end, while also using Bulma to style and create a simple, minimalistic look to the website. HTML/CSS, SCSS, JavaScript, React, Webpack, Bulma

Tech Used: HTML/CSS, SCSS, JavaScript, React, Webpack, Bulma

Project 3: City Spot

After eight weeks into the course, I began work on a group project with three others, over the course of one week we created a social media site for travellers to share photos and make posts from special moments during their travels. Two members of the group worked on the back-end, while I worked on the front-end with the other member. We used React to build out the front end, while using Semantic UI’s component library. On the back-end we used the MVC design pattern with NodeJS, Express, and MongoDB.

Tech Used: HTML/CSS, SCSS, JavaScript, React, Webpack, Semantic UI React, NodeJS, Express, MongoDB

Project 4: Read-Me

My fourth and final project of the course, on week 12, was also accomplished in a group, with two others. Over seven days we designed a mobile oriented web app, called Read-Me. It is a content creation site for authors and readers to share, read, and discover new short stories. We also included the Reddit API to pull short stories from an external source. For the first several days we worked all together on the backend, using Python, Flask, and SQLAlchemy to run the server and interact with the PostgreSQL database, again using the MVC design pattern. Then we transitioned to the front-end. On the front-end, I worked on creating the Reader page, the discover page, and setting up other various features such as night mode.

Tech Used: HTML/CSS, SCSS, JavaScript, React, Webpack, Semantic UI React, Python, Flask, SQLAlchemy, PostgreSQL
